Payu is an One Click Indian Payment Gateway Integration Kit for Laravel 5.2 and above.

Install Now for Free!

Download or Intall Payu for Laravel

We recommend installing through Composer.

View on GitHub Download .ZIP Download .TAR.GZ

Pull this package through Composer: 


$ composer require tzsk/payu

Configure Payu for Use

After installation configure it to suit your needs!

Add the Service Provider and Facade:

Go to config/app.php and add the following


'providers' => [
    ...
    Tzsk\Payu\Provider\PayuServiceProvider::class,
    ...
],
'aliases' => [
    ...
    'Payment' => Tzsk\Payu\Facade\Payment::class,
    ...
],

Once done run Artisan command:


$ php artisan vendor publish

Now you will have a file config/payu.php & a migration file for payu_payments table
Config file is where you have to put your creadentials. By default test creadentials are set. So you will have no problem testing.

Migrate Helper Table (Optional):


$ php artisan migrate

After that you will have a payu_payments table in your database. This is just a helper table, no need to change anything.

If you don't want to use the migration table then you can set the 'driver' => 'session' in the config file.

How to use Payu

Follow these simple steps and you are good to go!

Create the Endpoints:

First off you are going to need 2 endpoints (routes). One for Making the payment call and the other for recieving the payment status.

For example purposes I will make some assumptions


Route::get('/pay', ['as' => 'pay', 'uses' => 'PaymentController@pay']);

# You will need one more.

Route::get('/payment/status', ['as' => 'payment_status', 'uses' => 'PaymentController@status']);

/**
 * Using Named Routs to demonstrate all the possibilities.
 */

Create Controller Methods:

Use the Payment facade namespace at the top.


...
use Tzsk\Payu\Facade\Payment;
...

Make the Payment:


public function pay() {
    /**
     * These are the minimum required fieldset.
     */
     $data = [
         'txnid' => strtoupper(str_random(8)), # Transaction ID.
         'amount' => rand(100, 999), # Amount to be charged.
         'productinfo' => "Product Information",
         'firstname' => "John", # Payee Name.
         'email' => "john@doe.com", # Payee Email Address.
         'phone' => "9876543210", # Payee Phone Number.

         ... # Additional Fields With Data if any.
         ... # Optional Fields With Data if any.
     ];

     return Payment::make($data, function ($then) {
         $then->redirectTo('/payment/status');
         # OR...
         $then->redirectRoute('payment_status');
         # OR...
         $then->redirectAction('PaymentController@status');
         /**
          * Above are general Redirect::to(), Redirect::route() and Redirect::action() Methods.
          * You can use them as you normally would (With Parameters if you like) in any place to redirect.
          *
          * Note: You have to return the Payment Facade.
          */
     });
}

Additional Fields that can go with the $data


$additional_fields = [
    "lastname", "address1", "address2", "city", "state", "country", "zipcode"
];

Optional Fields that can go with the $data


$optional_fiels = [
    "udf1", "udf2", "udf3", "udf4", "udf5"
];

Recieve The Payment:


public function status() {
    $payment = Payment::capture(); # capture the payment after it's done. That's it.
    /**
     * $payment is the Model instance of payu_payment row.
     */
}

Payment API:


$payment->getData(); # Get the full response from Gateway.
$payment->isCaptured(); # Is the payment captured or some internal failure occured.
$payment->transaction_id; # Your Local Transaction ID.
$payment->payment_id; # PayU Payment ID.
$payment->total_amount; # Get Tototal Amount Deducted.
$payment->bank_reference_number; # Issued Bank Refernce Number.
$payment->bank_code; # Issued Bank Code.
$payment->card_number; # Redacted Card Number. If paid through Card.

$payment->get('attribute'); # Attribute is anything that Payu Post which returns in $payment->getData().
# Ex.
$payment->get('udf5');
# OR...
$payment->get('zipcode');

...
# And many more like 'status', 'mode' are found in the Database Table.

Advanced Feature:

Let's assume you have an Order Model and have a Subscription Model, and you are accepting paymets for both of them. Laravel Payu package makes it a breeze. You can have any number of Models that accept payment. To take advantage of the feature follow the below steps.

In your Models:


# Use the Fragment at the top.
...
use Tzsk\Payu\Fragment\Payable;
...

# Then inside the model class use the Fragment (Trait)

class Order extends Model {
    use Payable;
    ...
}
# That's it. Same applies to Any number of Models that you feel accept payments.

Now Change the Payment::make() slightly


# Instead of using Payment::make() directly use:
return Payment::with($order)->make(function ($then) {
    ...
    # Same as previous example.
});

# The $order variable is a placeholder that represents a perticular row instance of Any Model.
